### p.10 — 튜토리얼 NPC 설정 및 대사
- **[확정]** 동료 1~3에 자유 대화를 시도할 수 있는지 애매함(p.5에는 "자유 대화 연습"이 있는데 표에는 고정 대사만 있음). 아래 한 줄을 표 하단에 추가:
> ※ 동료 1~3은 자유 대화 입력이 가능하지만, 어떤 질문을 해도 자신의 힌트 범위 안에서만 대답한다 (튜토리얼 난이도 보호).

### p.11 — 튜토리얼 NPC 예외 상황
- **[확정]** 신뢰도 차감 트리거가 미정의였음 (어떤 실패가 누구의 신뢰도를 깎는지). 교체 문안:
> - 튜토리얼은 NPC별 신뢰도 2로 설정
> - **간부에게 접선 코드 입력 실패 시, 동료 전원의 신뢰도 -1**
> - 신뢰도 0이 된 NPC는 자기가 해당 단어를 떠올린 이유를 말한다 (접선 코드 직접 언급 X)
>   Ex) 접선 코드: 사과 / 단어: 빨강 → "이것의 색이 빨강이야."
> - **신뢰도 0 상태에서 한 번 더 실패하면**(누적 3회) 간부 앞으로 이동해 대화 연출
> - 간부와 대화 종료 시, **새로운 접선 코드로 교체 + 신뢰도 2로 리셋** (이전 힌트는 무효가 됐다는 안내 대사 출력)
- 사유: 원안대로 "2회 실패 → 전원 신뢰도 0 → 즉시 코드 리셋"이면, 신뢰도 0의 강화 힌트를 써볼 기회가 없이 리셋된다. 힌트 단계(2회 실패)와 리셋 단계(3회 실패)를 분리해야 두 장치가 모두 살아난다.

### p.16 — STAGE 1 위험 요소
- **[수정]** 감옥 퍼즐 실패 페널티 상충 해소(p.18은 "수류탄 차감"이라고 되어 있음): **실패 = 경비 레벨 +1**로 통일. 수류탄은 도주 미니게임 전용 자원으로 유지.
- **[확정]** 구출 성공 보상 추가 (현재 구현된 서버 로직과 동일):
> ● 성공 : 동료 구출 — **구출된 두 동료는 서로 겹쳐서 위험해졌던 '그 연상 단어'를 알려준다** (두 명이 같은 단어를 떠올렸다 = 코드와 강하게 연결된 단어 → 최상급 단서)
> ● 실패 : 경비 레벨 +1
- 사유: 쪽지(p.14)의 "같은 단어를 떠올렸다면, 좋은 단서가 될 수도…"가 이 보상의 복선. 보상이 명시되지 않으면 리스크(경비 레벨 +1)만 있는 선택지가 되어 아무도 구출을 시도하지 않는다.
- **[확정]** 하단에 한 줄 추가:
> ※ 수류탄·도주 미니게임은 스테이지 1 전용. 스테이지 3는 발각 게이지 방식(p.28)으로 동작한다.

### p.22 — 동료 식별 방식 상세
- **[확정]** 확인 테스트 규칙 통일: "(끝말잇기 5회 이상)" → "**(끝말잇기 — 5턴 이상 이어가면 통과)**"
- **[확정]** 동료 의심도 최대 도달 시의 결과가 미정의였음. 추가 문안:
> ▪ 동료 의심도 최대 도달 → 해당 동료는 경계하며 대화를 중단. **다른 NPC와 1회 이상 대화한 뒤 재시도 가능**하며, 호감도는 0부터 다시 쌓아야 한다 (게임오버 아님)
- **[제안]** 대안: 해당 동료가 방을 옮겨 잠적(찾는 수고 페널티). 동료 3명이 전원 필수(정보+열쇠)라서 영구 잠적은 소프트락이 되므로 금지.
- **[확정]** 호감도·의심도는 **NPC별 개별 카운트**다 (저택 전체 합산 아님). "민간인 의심도 최대(3회)"도 NPC별 3회.
- **[제안]** 민간인 대화에 중립 보상 배치: 안전한 화제에는 민간인도 방 배치·경비 공백·동료 소재의 옅은 힌트를 준다. 민간인 대화가 순수 리스크면 최적 행동이 "민간인 무시"가 되어 민간인 NPC 5명이 지뢰 장식이 된다.

## 3. 미결 사항 (팀 결정 필요)

1. **심문 질문 횟수 15회** — 목업 기준으로 확정했으나, 실플레이 테스트에서 페이싱 보고 조정 (긴장감이 부족하면 10회도 검토).
2. **동료 의심도 페널티** — 본 문서는 "대화 일시 중단 + 호감도 리셋"으로 확정. 더 강한 긴장감을 원하면 '방 이동(잠적)' 대안 검토. 단, 동료 3명이 전원 필수라 영구 잠적은 소프트락이므로 금지.
3. **스토리보드 ↔ 기존 계획서/구현의 관계 정리** — [NAN2026_계획서.md](NAN2026_계획서.md)와 현재 구현(스테이지 1)은 "동료 신뢰도 3칸 + 밀고 + 경계 레벨" 구조인데, 이 스토리보드의 스테이지 1에는 신뢰도·밀고 언급이 없다. 스테이지 1에 신뢰도 시스템을 유지할지(구현 유지), 스토리보드대로 뺄지 결정 필요. 유지한다면 p.16 위험 요소에 "코드 입력 실패 → 신뢰도 하락 → 밀고" 항목을 추가해야 한다. → **결정 (2026-07-21): 스테이지 1 신뢰도 제거.** 코드 오답 = 경계 +1 로 전환, 구현 반영 완료. p.16 위험 요소에는 "코드 오답 → 경계 +1" 로 기재.
4. **경비 레벨 초기값** — 명시된 곳이 없음. 초기값 1, 최대 3으로 가정하고 진행 중 (감옥 퍼즐 실패로만 상승). → **결정 (2026-07-21): 초기 0 · 상한 3 확정.** 상승 요인: 코드 오답·구출·자물쇠 실패·검문 적발 (각 +1).
5. **작품명** — 계획서는 '코드네임: 태엽새', 스토리보드는 'HEART OF STEEL (가제)'. 제출물 5종에 일관 표기 필요.
6. **심문 단어 카테고리** — "신분으로 연기 가능한 것(직업·역할)"으로 한정 (계획서 §4.5 확정). 구체 단어 풀(시계공/요리사/청소부 등)은 W3 심문 구현 시 확정.

## 4. 계획서 동기화 중 추가 발견 사항 (2026-07-21 추가)

[NAN2026_계획서.md](NAN2026_계획서.md)를 스토리보드 기준으로 동기화하면서 **구현 코드와 대조**해 새로 발견한 차이. 계획서 §4.6 "스토리보드 ↔ 현재 구현 조정 목록"에 조치 방안과 함께 정리했고, 여기에는 스토리보드 쪽에 반영이 필요한 것만 요약한다.

1. **감옥 퍼즐 방식 불일치** — 스토리보드(p.18): 3×3 숫자 순서 클릭·15초 / 구현(`lockPuzzle.js`): 사고형 퍼즐 3유형 랜덤 출제, 오답 2회 허용, LLM 미사용. 구현 쪽이 반복 출제 다양성과 소프트락 방지 설계가 우수하므로 **스토리보드 p.18을 구현 기준으로 역수정 권장**.
2. **스테이지 1 발각 대응 불일치** — 구현: 발각 → 검문(1단 타이밍 스캔 `timingLock.js` → 실패 시 2단 LLM 심문 `interrogation.js`) / 스토리보드: 발각 → 도주 QTE(수류탄). 스토리보드대로 가면 스테이지 1에서 LLM 심문이 사라지는 대신 스테이지 3 심문으로 이동하는 구조 — **기존 2단 검문 파이프라인은 폐기하지 말고 스테이지 3 심문의 기반으로 이관**.
3. **경비 레벨 상한 불일치** — 구현: 상한 4 (`MAX_LEVEL = 4`, Patrol·timingLock 공유) / 스토리보드: 상한 3 (레벨별 효과 정의됨). 스토리보드 기준 3으로 코드 조정 필요.
4. **체포 인원: 고정 vs 가변** — 스토리보드는 "동료 2명 구속"으로 고정 서술하지만, 구현·계획서는 중복 판정에 따라 0~4명 가변(리플레이성의 핵심). **가변 유지가 맞고, 스토리보드의 '2명'은 대표 연출 예시로 명시**할 것. 까마귀 쪽지·마을 사람 대사도 체포 인원에 맞춰 가변 문구로.
5. **접선 코드 입력 진입점** — 구현: 아무 동료 앞에서나 입력 가능 / 스토리보드 확정: 접선책(동료 5, 시계 수리공) 전용. 코드 수정 필요. → 반영 완료 (2026-07-21).
6. **구출 시 경비 레벨 +1 (구현에만 존재)** — `rescueAlly`는 구출 성공 시에도 경비 레벨을 1 올린다 ("구출 소동" 컨셉). 스토리보드 p.16에는 실패 페널티만 있음. 유지한다면 스토리보드에 "성공 시에도 소동으로 경비 레벨 +1" 명시, 폐기한다면 코드 제거 — **W2 초 결정 필요**. → **결정 (2026-07-21): 유지.** 스토리보드 p.16에 "구출 성공 시에도 소동으로 경계 레벨 +1" 명시 추가.
7. **동료 인원 구성 확정 제안** — "연상 단어 제출 동료 5명(중복 시 체포) + 접선책 1명(단어 제출 안 함)". 이 기준으로 p.13 맵 라벨(동료 NPC 수)과 p.19 표(동료 1~5 + 접선책)의 인원 표기를 한 번 정리해야 한다. 현재 스토리보드는 "동료 5명 + 감옥 2명"이 별도 인원인지 포함 인원인지 모호. → 구현 반영 (2026-07-21): 동료 5(단어 제출) + 접선책 요른(fixer, 단어 미제출) 확정.
