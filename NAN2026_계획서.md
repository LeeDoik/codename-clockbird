### 핵심 차별점 — "생성형 AI가 매 판 새로운 퍼즐을 만든다"

보드게임 **저스트원(Just One)** 의 핵심 메커니즘(제시어 → 각자 연상 단어 제출 → **중복 단어는 제거**)을 싱글플레이 게임으로 옮기되, 사람 플레이어 대신 **생성형 AI NPC가 연상 단어를 생성**한다.

- 접선 코드(제시어)와 동료 NPC들의 연상 단어가 **매 판 LLM에 의해 새로 생성** → 정해진 정답이 없는 무한 리플레이성
- 저스트원의 "중복 단어 제거" 규칙을 **"같은 단어를 떠올린 동료는 적에게 잡혀갔다"** 는 서사로 재해석 → 룰이 곧 스토리가 되는 구조
- NPC와의 자유 대화, 불심검문의 자유 입력 심사 등 게임플레이 전반에 AI가 개입 — "AI를 얹은 게임"이 아니라 **AI 없이는 성립하지 않는 게임**

## 3. 세계관·스토리

**배경**: 증기와 태엽의 도시. 스팀펑크 세계관 (레퍼런스: 네이버 웹툰 《마이너스의 손》 풍 비주얼).

- **지배 세력**: 로봇 + 소수 인간 협력자로 구성된 통치 기구. 도시를 감시망으로 통제
- **저항 세력**: 인간만으로 구성된 지하 조직. 플레이어는 이 조직의 신입 공작원
- **로봇 2종 설정** — 게임 시스템과 1:1 대응:
  | 설정 | 시스템 구현 |
  |---|---|
  | 명령 수행형 로봇 | 스크립트 기반 기존형 NPC (순찰 경로, 고정 패턴 — 몬스터 역할) |
  | 자율 사고형 로봇 | **생성형 AI 탑재 NPC** (자유 대화, 판단, 심문) |

**Stage 1 시놉시스**: 플레이어는 지배 세력 간부의 저택 구역에 잠입해, 흩어진 동료 5명과 접선하여 **최종 접선 코드**를 알아내야 한다. 동료들은 발각 위험 때문에 코드를 직접 말하지 못하고, 코드에서 **연상되는 단어 하나**만 암호처럼 흘린다. 같은 단어를 떠올린 동료들은 이미 패턴이 노출되어 잡혀간 상태다.

## 4. 핵심 게임플레이

### 4.1 게임 루프 (Stage 1)

```
스테이지 시작
 → [서버] 접선 코드 무작위 선정 + 동료 5명의 연상 단어 AI 생성
 → [서버] 중복 단어 판정 → 중복자는 '체포됨' 처리 (감옥 배치)
 → 플레이어: 맵 탐색 → 마을 NPC 대화(단서) → 동료 NPC 대화(연상 단어 수집)
 → (선택) 잡힌 동료 구출 → 새 연상 단어 획득
 → 접선 코드 입력 시도
     ├ 성공 → Stage 1 클리어
     └ 실패 → 해당 동료 신뢰도 -1 (0이 되면 밀고 → 경계 레벨 상승)
 → 순찰 NPC 감지 시 불심검문 (실패 시 게임오버 / 밀고 상태면 즉시 게임오버)
```

### 4.2 규칙 상세 (스케치 구체화)

**접선 코드 선정**
- 스테이지 시작 시 서버가 카테고리별 단어 풀(JSON, 예: 사물/장소/자연/개념 각 25개 내외)에서 무작위 1개 선정
- 코드 단어는 **서버만 보관** — 클라이언트에 전송하지 않음 (개발자 도구 치팅 방지, 정답 판정은 서버에서 수행)

**동료 NPC 5명 — 연상 단어 생성**
- 각 동료는 고유 페르소나 보유 (예: 시계공, 주방 하녀, 기관사, 밀수꾼, 거리 악사 — 직업·성격·말투 정의)
- 스테이지 시작 시 LLM이 "코드 단어를 보고 이 인물이 떠올릴 법한 연상 단어 1개"를 페르소나별로 생성
- 페르소나가 다르므로 연상 방향이 자연스럽게 갈라짐 → 단서의 다양성 확보

**중복 판정 (저스트원 규칙의 서사화)**
- 1차: 표기 정규화(공백·기호 제거, 대소문자 통일) 후 완전 일치 검사
- 2차: LLM 유사어 판정 — 동의어·표기 변형(예: "톱니바퀴"/"기어")을 중복으로 간주할지 structured output으로 판정
- 중복된 동료들은 **체포됨** 상태로 감옥 셀에 배치, 맵의 접선 지점에서 제외

**신뢰도 시스템**
- 동료당 신뢰도 3칸 (UI에 표시)
- 플레이어가 그 동료 앞에서 접선 코드 입력에 실패하면 해당 동료 신뢰도 -1
- 신뢰도 0 → 동료가 플레이어를 **밀고**하고 맵에서 사라짐 → 저택 전체 **경계 레벨 상승**
- 밀고 상태에서 순찰 NPC에게 감지되면 검문 없이 **즉시 게임오버**

**접선 코드 입력·판정**
- 아무 동료 NPC 앞에서나 시도 가능, 텍스트 자유 입력
- 판정: 정규화 후 코드 단어와 일치하면 성공. 경계 사례(복수 표기, 유의어)는 LLM 판정으로 보완 — "사실상 같은 단어"만 허용, 상위/연관 개념은 불허
- 성공 시 즉시 스테이지 클리어

**적 NPC ① — 감시 로봇 (잡힌 동료를 감시)**
- 대화 시도 → 간단한 미니게임 진입 (개발 리스크 최소화를 위해 **QTE(타이밍 바) 또는 선택지형 중 W2에 1개 확정**)
- 실패 → 게임오버 / 성공 → 동료 구출 → 해당 동료는 **새로운 연상 단어**(기존 배치 단어들과 중복되지 않게 재생성)를 갖고 맵에 재배치

**적 NPC ② — 순찰 로봇 (불심검문)**
- 시야/감지 범위 존재 (시각화). 진입 시 불심검문 시작
- 검문 = **선택지 + 자유 입력 하이브리드**: 로봇이 심문 질문을 생성하고, 플레이어는 준비된 선택지 또는 자유 입력으로 답변 → LLM이 답변의 '수상함'을 심사 (structured output: 통과/의심/적발)
- 적발 시 게임오버. 자유 입력 심문은 본 게임의 대표적인 AI 어필 포인트

**마을 시민 NPC**
- 자유 대화형 (생성형 AI). 역할: ① 세계관 설명 ② 동료 NPC 위치 단서 ③ 현재 체포 상황 브리핑
- 게임 상태(체포된 동료 수 N)가 시스템 프롬프트에 주입되어 대사가 상황에 따라 변화

### 4.3 예상 시나리오 (스케치 원안 반영)

1. **튜토리얼**: 저항 세력 간부에게 임무 브리핑을 받고 규칙(연상 단어, 신뢰도, 검문) 숙지 → 미션 장소로 이동
2. **미션 초입**: 먼저 말을 거는 마을 NPC 배치 — 체포 상황에 따라 분기:
   - 동료가 한 명도 안 잡힘: *"요즘은 저항 세력이 잡혔다는 소식이 잘 안 들려서 다행이야…"* → 나레이션: *"5명 모두 무사해서 다행이야…"*
   - N명 잡힘: *"요즘 경비가 한층 삼엄해졌어. 얼마 전에 저항 세력이 N명이나 잡혔다지…"* → 나레이션: *"그럼 (5-N)명뿐인가… 일단 잡히지 않은 동료부터 찾아보자."*
   - 5명 전원 체포 (예외): *"모두 잡혔잖아… 위험하지만 동료부터 구해야겠어."* → 구출이 선행 과제가 됨
3. 이 정보를 바탕으로 플레이어가 **탐색 우선 / 구출 우선** 플레이 방향을 스스로 선택
4. 단어 수집 → 추리 → 코드 입력 → 클리어 (또는 신뢰도 관리 실패·검문 적발로 게임오버)

## 5. AI 시스템 설계 (AI 활용 기술 문서의 뼈대)

### 5.1 AI 활용 지점 요약

| # | 기능 | 모델 | 호출 시점 | 방식 |
|---|---|---|---|---|
| 1 | 연상 단어 생성 (동료 5인) | `claude-sonnet-5` | 스테이지 시작 시 1회 (일괄) | structured outputs |
| 2 | 중복·유사어 판정 | `claude-sonnet-5` | 단어 생성 직후 / 코드 입력 시 | LLM-as-judge, structured outputs |
| 3 | NPC 자유 대화 (마을·동료·간수) | `claude-haiku-4-5` | 플레이어 대화 시 실시간 | 페르소나 시스템 프롬프트 + 프롬프트 캐싱 + 스트리밍 |
| 4 | 불심검문 심문·심사 | `claude-sonnet-5` | 검문 이벤트 시 | 질문 생성 + 답변 심사 (structured outputs) |

모델 역할 분담 원칙: **실시간 대화는 저지연·저비용 Haiku, 게임 룰에 영향을 주는 생성·판정은 Sonnet.**

### 5.2 파이프라인 설계

**① 연상 단어 생성** (스테이지 시작 시 서버에서 실행)
```
입력: 코드 단어 + 동료 5인 페르소나 정의
출력(JSON 스키마 강제): [{ npcId, word, reason }, ...]
제약: 코드 단어 자체·코드 단어를 포함한 합성어 금지, 한 단어(명사)만 허용
→ 정규화 후 중복 검사 → 중복 그룹은 '체포' 처리
→ 결과를 서버 세션에 저장 (이후 게임 내내 재사용, 추가 호출 없음)
```

**② NPC 자유 대화** (실시간)
```
시스템 프롬프트 = [세계관 공통 블록(캐싱)] + [페르소나 블록(캐싱)] + [현재 게임 상태(체포 수, 경계 레벨 등)]
가드레일: "접선 코드 단어를 절대 직접 말하지 않는다" + 서버측 출력 필터
         (응답에 코드 단어 포함 시 폐기 후 재생성, 최대 2회 → 실패 시 회피 대사 폴백)
스트리밍 응답 → 대화창에 타자 효과로 표시 (체감 지연 최소화)
```

**③ 검문 심사**
```
심문 질문: 로봇 페르소나가 상황(장소, 경계 레벨) 기반으로 생성
플레이어 답변(자유 입력) → LLM 심사 → { verdict: "pass" | "suspicious" | "caught", npcReply }
suspicious 시 추가 질문 1회 (긴장감 연출), caught 시 게임오버
```

### 5.3 프롬프트 설계 예시 (발췌)

동료 NPC 연상 단어 생성 (시스템 프롬프트 요지):
```
너는 스팀펑크 도시의 저항 세력 조직원 5명이다. 각자 아래 페르소나를 가진다. (…페르소나 5개…)
비밀 접선 코드는 "{codeWord}"다. 각자 자신의 직업과 경험에 비추어
이 단어에서 연상되는 '한 단어'를 하나만 고른다.
규칙: 코드 단어와 같거나 코드 단어를 포함한 단어는 금지. 반드시 지정된 JSON 스키마로만 응답.
```

### 5.4 비용·지연 추정 (사전 과제 개발 + 시연 기준)

- 대화 1턴 (Haiku 4.5, $1/$5 per MTok): 입력 ~1.5K + 출력 ~0.2K 토큰 ≈ **$0.003 미만** — 캐싱 적용 시 그 이하
- 단어 생성·판정 (Sonnet 5, 인트로가 $2/$10 per MTok, ~2026-08-31): 스테이지당 2~4회 호출 ≈ **$0.02 미만**
- 개발 + 테스트 + 시연 전 기간 합산해도 수 달러 수준 → 비용 리스크 낮음
- 지연 대응: 단어 생성은 로딩 화면 뒤로 숨김(1회성), 대화는 스트리밍, 판정은 짧은 연출(로봇 "분석 중…")로 흡수

### 5.5 리플레이성·심사 어필 포인트

- 코드 단어 100개 풀 × 페르소나별 연상 생성 → **같은 판이 반복되지 않음**
- 대화가 스크립트가 아니므로 시연 중 심사위원이 직접 자유롭게 질문 가능 → 라이브 데모에 강함
- "룰(중복 제거)이 서사(체포)가 되는" AI-네이티브 게임 디자인을 기술 문서에서 명시적으로 서술

## 6. 기술 아키텍처

```
[브라우저]                         [Node.js 서버]                [Claude API]
 Phaser 3 게임 클라이언트  ⇄ REST/  Express                    ⇄  @anthropic-ai/sdk
  - 톱다운 맵/이동/시야      SSE     - API 키 보호 프록시           - claude-haiku-4-5 (대화)
  - 대화 UI(스트리밍 표시)          - 게임 세션 상태 관리            - claude-sonnet-5 (생성/판정)
  - 신뢰도/경계 HUD                 - 코드 단어 보관·정답 판정
  - 미니게임                        - 가드레일 출력 필터
```

- **클라이언트**: Phaser 3 (2D 톱다운, 타일맵, 충돌·시야 감지). 대화 UI는 DOM 오버레이로 구현(한글 입력 편의)
- **서버**: Node.js + Express 단일 프로세스. 세션은 인메모리(싱글플레이·로컬 실행이므로 충분)
- **Claude 연동**: `@anthropic-ai/sdk` (TypeScript). structured outputs는 `client.messages.parse` + `zodOutputFormat`(Zod 스키마) 사용, 대화는 `client.messages.stream` → SSE로 클라이언트 중계
- **비밀 관리**: `ANTHROPIC_API_KEY`는 서버 `.env`로만 주입 — 브라우저에서 Claude 직접 호출 금지
- **정적 데이터**: 코드 단어 풀, NPC 페르소나, 맵/대사 스크립트 → JSON
- **빌드 제출 형태**: `npm install && npm start` → 브라우저 접속. Windows용 원클릭 실행 스크립트(`play.bat`)와 README(요구 사항: Node 22+, API 키 설정법) 동봉
