### 핵심 차별점 — "생성형 AI가 매 판 새로운 퍼즐을 만든다"

보드게임 **저스트원(Just One)** 의 핵심 메커니즘(제시어 → 각자 연상 단어 제출 → **중복 단어는 제거**)을 싱글플레이 게임으로 옮기되, 사람 플레이어 대신 **생성형 AI NPC가 연상 단어를 생성**한다.

- 접선 코드(제시어)와 동료 NPC들의 연상 단어가 **매 판 LLM에 의해 새로 생성** → 정해진 정답이 없는 무한 리플레이성
- 저스트원의 "중복 단어 제거" 규칙을 **"같은 단어를 떠올린 동료는 적에게 잡혀갔다"** 는 서사로 재해석 → 룰이 곧 스토리가 되는 구조
- 종반부 **심문 미니게임은 '이중 AI' 구조** — 플레이어의 비밀을 아는 시스템 AI(거짓 탐지)와 모르는 로봇 AI(추리·모순 탐지)가 동시에 판정 → "AI를 얹은 게임"이 아니라 **AI 없이는 성립하지 않는 게임**

## 3. 세계관·스토리 (스토리보드 p.2, p.35 기준)

**배경**: 강철이 지배하는 시대. 브루주아 계급이 강철 로봇으로 세계를 통제하는 디스토피아. (거리: 주황·베이지 톤 건물 + 스모그 낀 어두운 조명 / 본부: 따뜻한 조명으로 대비)

1. **브루주아의 지배** — 브루주아 계급이 강철 로봇으로 세계를 통제한다
2. **명령만 따르는 로봇** — 로봇은 명령 수행만 가능하며 사고 능력은 없다
3. **접선 코드** — 레지스탕스는 로봇이 이해할 수 없는 방식(단어 연상)으로 서로를 식별한다
4. **수상한 소문** — 최근 브루주아 저택에서 이상한 소문이 발생했다

- **로봇 2종 설정** — 게임 시스템과 1:1 대응:
  | 설정 | 시스템 구현 |
  |---|---|
  | 명령 수행형 로봇 (경비·순찰) | 스크립트 기반 기존형 NPC (순찰 경로, 시야 콘, 고정 패턴 — 몬스터 역할) |
  | 자율 사고형 신형 로봇 ('꼬마') | **생성형 AI 탑재 NPC** (자유 대화, 추리, 심문) — 종반 반전의 주역 |

**전체 흐름** (스토리보드 씬 구성):

| # | 씬 | 씬 목적 |
|---|---|---|
| 0 | 오프닝 시퀀스 | 세계관 분위기 전달 (자동 재생, 조작 없음) |
| 1 | 튜토리얼 · 레지스탕스 본부 | 조작·대화·접선 코드 풀이 방식 학습 |
| 2 | 스테이지 1 · 거리 | 연상 단어 수집 → 접선 코드 추리, 잠입 감각 전달 |
| 3 | 스테이지 2 · 저택 잠입 | 자유 대화로 동료 식별, 정보 수집, 신형 로봇 기록 발견 |
| 4 | 스테이지 3 · 저택 탈출 | 강화된 경비 회피 + 신형 로봇 '꼬마'와 심문 대결 |
| 5 | 엔딩 · 레지스탕스 본부 | 보고("자유의지를 가진 로봇"), 다음 이야기 암시 |

**주요 캐릭터**: 플레이어(레지스탕스 신입 공작원) / 간부(브리핑·보고) / 거리의 동료들(연상 단어 제공, 그중 시계 수리공이 저택 잠입의 연결 고리) / 저택 내부 동료들(정보 조각 전달) / 신형 인간형 로봇 '꼬마'(자유 의지를 지닌 미지의 존재)

## 4. 핵심 게임플레이 (스토리보드 + 수정안 확정 규칙 기준)

### 4.1 공통 시스템

- **대화 시스템**: NPC 일러스트(하단 좌측 대형) + 대사 바 + 우측 세로 아이콘 메뉴(대화하기/접선 코드/닫기, 2~3개 가변)
  - **일반 대화**: 입력된 대사 출력 (연출용) / **자유 대화**: 플레이어 텍스트 직접 입력 → 생성형 AI 응답
- **경비 레벨**: 1~3단계. 2: 경비 로봇 추가 배치 + **도주 QTE 제한 시간 단축(15→10초)** / 3: 이동 속도·탐지 범위 증가 + **발각 시 도주 미니게임 없이 즉시 게임오버** — 진입 시 "자석이 통하지 않는 신형 경비 투입" 연출로 수류탄 무력화를 정당화 (수류탄이 남았는데 QTE 없이 즉사하는 상황의 직관 보정)
  - 레벨을 QTE 난이도에도 연동한다 — 기존 구현에서 경계 레벨이 검문 심사 엄격도까지 조이던 "두 번째 이빨"을 QTE 체계에서도 유지 (연동이 없으면 레벨 1↔2의 체감 차이가 로봇 수뿐이 된다)
  - (밸런스 테스트 항목: 1분간 발각되지 않으면 1단계 하강 규칙 적용 검토)
- **게임오버 공통 연출**: "강철의 신념은 꺾이지 않습니다…" → 다시하기 시 해당 스테이지 재시작 (리로드 없음)

### 4.2 튜토리얼 — 레지스탕스 본부

- 간부 브리핑 → 동료 1~3과 대화(색상/분류/대중적 특징 힌트) → 간부에게 접선 코드 입력 → 성공 시 스테이지 1
- **신뢰도 규칙 (확정)**: NPC별 신뢰도 2. 코드 입력 실패 시 동료 전원 -1 → 0이 되면 각자 "단어를 떠올린 이유"를 직접 설명(강화 힌트) → 0 상태에서 재실패(누적 3회) 시 간부가 **새 코드로 교체 + 신뢰도 리셋**
- 동료 1~3은 자유 대화 입력이 가능하되 힌트 범위 안에서만 응답 (난이도 보호)

### 4.3 스테이지 1 — 거리 (핵심 루프)

```
스테이지 시작
 → [서버] 접선 코드 무작위 선정 + 동료들의 연상 단어 AI 생성
 → [서버] 중복 단어 판정 → 중복자는 '체포됨' 처리 (감옥 배치, 0~N명 가변)
     ※ 스토리보드의 "2명 구속"은 대표 연출 예시 — 실제는 매 판 가변 (까마귀 쪽지 문구도 인원에 맞춰 가변)
 → 플레이어: 9섹터 맵 탐색 → 마을 사람 대화(단서) → 자유 동료 대화(연상 단어 수집)
 → (선택) 감옥 구출 — 잠금 퍼즐: 성공 = 동료 구출 + 겹쳤던 연상 단어 획득(최상급 단서) / 실패 = 경비 레벨 +1
 → 경비 로봇에게 발각 → 도주 미니게임(QTE): 성공 = 수류탄 회수(유지) / 실패 = 수류탄 차감, 0개면 게임오버
 → 접선책(시계 수리공)에게 접선 코드 입력 → 성공 시 클리어 → 저택 잠입 연결 연출
```

- **동료 구성 (확정 제안)**: 연상 단어 제출 동료 5명(중복 시 체포) + **접선책(시계 수리공) 1명** — 코드 입력은 접선책에게만 가능
- **도주 미니게임**: 발각 → 15초 제한 2단 QTE(연타로 게이지 → 거리 조절 타이밍 입력). 수류탄 2개 시작, "게임오버 = 자석 수류탄 소진" 컨셉. **수류탄·도주 QTE는 스테이지 1 전용**
- **감옥 구출 퍼즐**: 감옥 문 상호작용 → 잠금 퍼즐 (LLM 미사용 — 지연·장애 리스크 차단). 구출 보상은 저스트원 서사의 핵심: *두 명이 같은 단어를 떠올렸다 = 코드와 강하게 연결된 단어*
- 까마귀 쪽지: 시작 시 팝업 브리핑(체포 상황·주의사항) → 좌측 하단 메모 아이콘으로 상시 열람, 획득 단서 기록

### 4.4 스테이지 2 — 저택 잠입 (경비 없음 · 전원 자유 대화)

- 시계 수리공의 보조로 위장 입장. 방 단위 구획(홀·주방·서재·복도·세탁실·연구실)
- **동료 식별**: 직원 NPC와 자유 대화 — 반브루주아 발언 → 동료 호감도 ↑ / 민간인 의심도 ↑, 친브루주아 발언 → 동료 의심도 ↑(호감도 봉쇄) / 민간인 무변화. **수치는 UI 비노출, 대화 뉘앙스로 유추**. 호감도·의심도는 **NPC별 개별 카운트** (저택 합산 아님)
- **민간인 의심도 최대(NPC별 3회)** → 밀고 → 즉시 게임오버
- **민간인 대화 보상**: 민간인도 안전한 화제에는 중립 정보(방 배치·경비 공백 시간·동료 소재의 옅은 힌트)를 준다 — 민간인 대화가 순수 리스크면 최적 행동이 "민간인 무시"가 되어 NPC 5명이 지뢰 장식이 된다. 안전 화제로 탐색하고 확신이 설 때만 위험 발언으로 확인하는 리스크 관리 구조로 만든다
- **동료 의심도 최대** → 대화 일시 중단, 다른 NPC와 대화 후 재시도 가능(호감도 0부터)
- **확인 테스트**: 동료 호감도 임계 도달 → 끝말잇기 역제안, **5턴 이상 이어가면 통과** (실패 페널티 없음, 즉시 재도전)
- 동료 3명(수리공 제외)에게서 정보 조각 + 연구실 열쇠 획득 → 연구실 입장 → **신형 로봇 기록(문서) 열람 시 클리어** → 발각 소리에 문서를 챙겨 도주 연출 → 스테이지 3
- 동료 NPC는 첫 문장에 반브루주아 뉘앙스 힌트 배치

### 4.5 스테이지 3 — 저택 탈출 + 심문

- **잠입 파트**: 경비 시야 콘 확대 표시, 시야 진입 시 발각 게이지 상승, 엄폐물로 차단. **게이지 만충 = 즉시 게임오버** (도주 QTE 없음). 비밀 통로(지하, 파이프·나무 상자 엄폐물)로 최종 목적지 이동
- **심문 미니게임** (탈출 지점에서 꼬마 아이 조우 → "사실 나는 로봇이야" → 심문 개시):
  - 컨셉: *"선택한 가짜 신분을 끝까지 연기하라 — 진술이 흔들리는 순간 탐지된다."*
  - 심문 카테고리에서 무작위 단어 3개 생성 → 플레이어가 1개 선택(카드) → 이후 로봇의 질문에 **자유 입력**으로 답변. 단어 카테고리는 **신분으로 연기 가능한 것(직업·역할)** 으로 한정 — "선택 단어와 어긋난 거짓" 판정이 성립하려면 단어가 연기의 대상이어야 한다
  - **탐지 게이지 100 시작**, 감점 축:
    - 시스템 AI (선택 단어를 앎): ① 선택 단어와 어긋난 거짓 진술 **-20** ② 선택 단어를 **직접 노출**하는 진술 **-20** — 거짓도, 지나친 진실도 위험한 양면 긴장을 규칙으로 성립시킨다
    - 로봇 AI (선택 단어를 모름): 이전 답변과 **명시적으로 상충**하는 모순 — **첫 모순은 로봇이 지적하고 해명 기회 1회**(감점 없음, LLM 오탐 1회 흡수 + 긴장 연출), 이후 모순부터 **-50**
  - **로봇의 단어 추리는 "정식 추리 선언"으로만** (최대 2회): 답변 이력에서 내부 확신도를 축적하다 임계 도달 시 선언, 적중 판정은 정답 판정기(동의어 인정) 재활용. **로봇은 후보 3개를 모른다** — 알면 3지선다 소거법으로 필승이고, 모른 채 무제한 추리면 죽은 규칙이 되므로 "비공개 + 제한 선언"으로 양쪽 붕괴를 피한다
  - **패배**: 추리 선언 적중 or 게이지 0 (경비 호출 → 게임오버) / **승리**: 15문 방어 — "흥미롭네, 가도 좋아." (질문 횟수는 실플레이 페이싱에 따라 10회 하향 검토 — 수정안 §3)
- 엔딩: 본부 귀환 → 문서 전달 + "자유의지를 가진 로봇" 보고 → "반격의 서막" → To be continued

### 5.1 AI 활용 지점 요약

### 5.2 파이프라인 설계

**① 연상 단어 생성** (스테이지 시작 시 서버에서 실행)
```
입력: 코드 단어 + 동료 5인 페르소나 정의
출력(JSON 스키마 강제): [{ npcId, word, reason }, ...]
제약: 코드 단어 자체·코드 단어를 포함한 합성어 금지, 한 단어(명사)만 허용
→ 정규화 후 중복 검사 → 중복 그룹은 '체포' 처리
→ 결과를 서버 세션에 저장 (이후 게임 내내 재사용, 추가 호출 없음)
```

**② NPC 자유 대화** (실시간 — 구현 완료분 기준으로 기술)
```
시스템 프롬프트 = [세계관 공통 블록] + [페르소나 블록] + [현재 게임 상태(체포 수, 경비 레벨 등)]
가드레일 = 필터가 아니라 정보 설계: 대화 모델의 프롬프트에 접선 코드를 아예 넣지 않는다.
         모델이 모르는 정보는 유출될 수 없다 — 스트리밍에서는 출력 필터가 감지한 시점에
         이미 글자가 화면에 찍혀 있으므로 필터+재생성은 스트리밍 경로에 무력하다.
         필터+재생성은 코드를 알아야만 하는 파이프라인(① 단어 생성)에만 적용한다.
         wordGen 의 reason(코드를 알고 쓴 문장)도 대화 프롬프트에 절대 넣지 않는다.
스테이지 2 확장: 성향 판정 { affinityDelta, suspicionDelta } 는 대화 스트리밍과 분리된
              별도 Haiku 호출로 수행 (structured, 응답 직후 비동기).
              동일 호출로 합치면 응답 전체가 JSON 스트림이 되어 타자 효과에 증분 파싱이
              필요해진다 — 수치 비노출이라 판정이 수백 ms 늦어도 무방하므로 분리가 정답.
스트리밍 응답 → 대화창에 타자 효과로 표시 (체감 지연 최소화)
```

**③ 심문 이중 AI** (스테이지 3 — 본 게임의 대표 AI 어필 포인트)
```
구조: 서로 정보량이 다른 두 판정자가 동시에 작동
 - 시스템 AI  : 플레이어가 선택한 단어를 안다 → 거짓 진술(-20)과 단어 직접 노출(-20)을 판정
 - 로봇 NPC_AI: 선택 단어를 모른다 → 질문 생성(Haiku) + 답변 이력에서 명시적 모순 탐지
               (첫 회는 지적+해명 기회, 이후 -50) + 내부 확신도 축적 → 임계 도달 시
               정식 추리 선언 (최대 2회, 적중 판정은 정답 판정기의 동의어 판정 재활용)
턴 흐름: 로봇 질문 생성 → 플레이어 자유 입력 → [병렬] 시스템 AI 판정 + 로봇 AI 모순·확신도 갱신
종료: 추리 선언 적중 or 게이지 0 → 패배 / 15문 방어 → 승리
설계 의도: "완전한 거짓말(들통)도, 완전한 진실(단어 노출)도 안 되는" 긴장 — 거짓은 시스템 AI가,
          과도한 진실은 노출 감점과 로봇의 확신도 축적이 처벌한다. LLM이어야 가능한 판정
기반: 기존 검문 파이프라인(checkpoint.js: 질문 생성 + 답변 심사)을 확장 이관.
     단 검문은 단발 심사, 심문은 이력 누적·확신도 추적이 필요해 실질 신규 개발량이 크다
     — "이관"이라는 말의 착시를 경계하고 W3 최우선으로 착수한다
```

### 5.3 프롬프트 설계 예시 (발췌)

동료 NPC 연상 단어 생성 (시스템 프롬프트 요지):
```
너는 강철 로봇이 지배하는 도시의 레지스탕스 조직원 5명이다. 각자 아래 페르소나를 가진다. (…페르소나 5개…)
비밀 접선 코드는 "{codeWord}"다. 각자 자신의 직업과 경험에 비추어
이 단어에서 연상되는 '한 단어'를 하나만 고른다.
규칙: 코드 단어와 같거나 코드 단어를 포함한 단어는 금지. 반드시 지정된 JSON 스키마로만 응답.
```

### 5.4 비용·지연 추정 (사전 과제 개발 + 시연 기준)

- 대화 1턴 (Haiku 4.5, $1/$5 per MTok): 입력 ~1.5K + 출력 ~0.2K 토큰 ≈ **$0.003 미만** (성향 판정 분리 호출을 더해도 2배 이내)
- 단어 생성·판정 (Sonnet 5, 인트로가 $2/$10 per MTok, ~2026-08-31): 스테이지당 2~4회 호출 ≈ **$0.02 미만**
- 심문 1판 (질문 생성 Haiku + 판정·추리 Sonnet, 최대 15문): ≈ **$0.10 미만** — 종반 1회성 이벤트라 총비용 영향 미미
- **프롬프트 캐싱은 현재 미적용이 정직한 서술**: Haiku 4.5 의 최소 캐시 프리픽스는 4096 토큰인데 대화 시스템 프롬프트는 ~500 토큰이라 `cache_control` 을 붙여도 조용히 무시된다 (구현에서 의도적으로 생략 — `dialogue.js` 주석 참조). 대화 이력이 길어지는 저택 파트에서 프리픽스가 4096 토큰을 넘기면 도입한다. 기술 문서에는 **usage 필드로 검증 가능한 주장만** 쓴다
- 개발 + 테스트 + 시연 전 기간 합산해도 수 달러 수준 → 비용 리스크 낮음
- 지연 대응: 단어 생성은 로딩 화면 뒤로 숨김(1회성), 대화는 스트리밍, 심문 판정은 로봇 "분석 중…" 연출로 흡수. **반복 빈도가 높은 미니게임(도주 QTE·감옥 퍼즐)은 LLM 미사용** — 지연·장애가 게임 흐름을 막지 않게 격리

### 5.5 리플레이성·심사 어필 포인트

- 코드 단어 100개 풀 × 페르소나별 연상 생성 → **같은 판이 반복되지 않음** (체포 인원·구도도 매 판 가변)
- 대화가 스크립트가 아니므로 시연 중 심사위원이 직접 자유롭게 질문 가능 → 라이브 데모에 강함
- "룰(중복 제거)이 서사(체포)가 되는" AI-네이티브 게임 디자인을 기술 문서에서 명시적으로 서술
- **심문 이중 AI**: 정보 비대칭이 있는 두 판정자 구조 자체가 기술 문서의 대표 다이어그램 소재 — "명령만 따르는 로봇 vs 사고하는 로봇"이라는 세계관 설정이 그대로 AI 아키텍처로 구현됨을 어필
- **가드레일도 정보 비대칭 설계**: 대화 모델은 접선 코드를 모른다 — "필터로 막는다"가 아니라 "모르는 것은 유출될 수 없다". 심문 이중 AI와 같은 원리를 방어에도 적용했음을 기술 문서의 한 축으로 서술 (중복·정답 판정도 1차 정규화 코드 판정 → 경계 사례만 LLM 인 하이브리드 — 지연·비용 설계의 근거로 함께 어필)

## 6. 기술 아키텍처

```
[브라우저]                         [Node.js 서버]                [Claude API]
 Phaser 게임 클라이언트    ⇄ REST/  Express                    ⇄  @anthropic-ai/sdk
  - 톱다운 맵/이동/시야      SSE     - API 키 보호 프록시           - claude-haiku-4-5 (대화·성향 판정)
  - 대화 UI(스트리밍 표시)          - 게임 세션 상태 관리            - claude-sonnet-5 (생성/판정/심문)
  - 경비 레벨/발각 게이지 HUD       - 코드 단어 보관·정답 판정
  - 미니게임(도주 QTE·감옥          - 가드레일 출력 필터
    퍼즐·끝말잇기·심문)
```

- **클라이언트**: Phaser (2D 톱다운, 타일맵, 충돌·시야 감지). 대화·미니게임 UI는 DOM 오버레이(`MinigamePanel`)로 구현 (한글 입력 편의)
- **서버**: Node.js + Express 단일 프로세스. 세션은 인메모리 (싱글플레이·로컬 실행이므로 충분)
- **Claude 연동**: `@anthropic-ai/sdk`. structured outputs는 `client.messages.parse` + `zodOutputFormat`(Zod 스키마) 사용, 대화는 `client.messages.stream` → SSE로 클라이언트 중계
- **비밀 관리**: `ANTHROPIC_API_KEY`는 서버 `.env`로만 주입 — 브라우저에서 Claude 직접 호출 금지
- **정적 데이터**: 코드 단어 풀, NPC 페르소나, 맵/대사 스크립트 → JSON
- **빌드 제출 형태**: `npm install && npm start` → 브라우저 접속. Windows용 원클릭 실행 스크립트(`play.bat`)와 README(요구 사항: Node 22+, API 키 설정법) 동봉

## 9. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| **범위 확장** (스토리보드가 3스테이지+엔딩으로 확대, 잔여 2.5주) | 미완성 제출 | 아래 **컷 우선순위** 사전 합의. 사수 라인: **튜토리얼 + 스테이지 1 + 심문 미니게임** |
| LLM 응답 지연으로 게임 흐름 끊김 | 체감 품질 저하 | 단어 생성은 로딩 뒤로 숨김(1회성), 대화 스트리밍, 심문은 질문 생성 Haiku + "분석 중" 연출로 흡수. 고빈도 미니게임은 LLM 미사용 설계 유지 |
| 부적절·스포일러 출력 (NPC가 코드 단어 발설 등) | 게임 파괴 | **정보 설계로 원천 차단**: 대화 모델 프롬프트에 코드를 넣지 않는다 (모르는 것은 유출 불가). 필터+재생성은 코드를 아는 파이프라인(단어 생성)에만 적용 |
| 심문 모순 판정 오탐 (-50 오폭) | 불합리한 패배 | 모순 기준을 "명시적 상충"으로 한정 + 첫 모순은 지적·해명 기회(감점 없음), 이후부터 -50 — 오탐 1회를 규칙이 흡수 |
| 연상 단어 품질 편차 (너무 쉽거나 뜬금없음) | 퍼즐 난이도 붕괴 | 프롬프트 튜닝(직접성 레버는 중복 폭등 부작용 확인됨 — 대화·판정 레버로 조정), 단어 풀 큐레이션, LLM 판정 난이도 필터 |
| 스테이지 2 성향 판정 오판 (민간인 억울한 밀고 등) | 불합리한 게임오버 | 판정을 대화 호출과 통합해 문맥 유지, 의심도 3회 누적제(1회 오판 허용), 경고성 대사 연출로 신호 제공 |
| 네트워크/API 장애 시 시연 불가 | 심사 리스크 | **데모 폴백 모드(계층형)**: 단어 생성·중복 판정은 사전 생성 세트 / 자유 대화는 고정 대사 모드로 강등 / 심문은 검문 1단처럼 선택지형으로 강등. 자유 입력은 원리상 캐시가 불가능하므로 "무엇이 꺼진 모드인지"를 미리 정의해야 시연 사고가 안 난다 (`checkpoint.js` 의 캔 질문 + fail-open 패턴 재활용) |
| 심사위원 빌드 실행 환경에서 API 키 문제 | 심사 불가 | README에 키 설정 1분 가이드 + 폴백 모드 안내, 시연 영상으로 보완 |

**컷 우선순위** (지연 시 위에서부터 순서대로 컷):
1. 스테이지 2 민간인 NPC 5명 → 3명 축소 (오프닝 시퀀스는 이미 구현되어 컷 이득이 없으므로 목록에서 제외)
2. 스테이지 3 잠입 구간 축소 → 비밀 통로 최소 맵, **심문은 유지**
3. 스테이지 2 끝말잇기 → 선택지형 확인 테스트로 단순화
4. 최후: 스테이지 2·3 전체 컷 → 심문을 스테이지 1 종반 이벤트로 이식 (사수 라인으로 후퇴)
