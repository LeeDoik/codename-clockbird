# 프롬프트 튜닝 가이드 (팀원용)

> 코드를 몰라도 됩니다. 브라우저 화면에서 NPC의 성격과 프롬프트를 고치고,
> 저장하기 전에 실제 AI 응답을 미리 확인할 수 있어요.

---

## 1. 처음 한 번만 — 환경 준비 (~10분)

1. **GitHub 초대 수락** — 이메일(또는 github.com 알림)에 온 `codename-clockbird` 저장소 초대를 수락
2. **Git 설치** — [git-scm.com](https://git-scm.com) (설치 옵션은 전부 기본값으로 Next)
3. **Node.js 22 이상 설치** — [nodejs.org](https://nodejs.org) (LTS 버튼)
4. **프로젝트 받기** — 터미널(PowerShell)을 열고:

   ```bash
   git clone https://github.com/LeeDoik/codename-clockbird.git
   cd codename-clockbird
   npm install
   ```

5. **API 키 설정** — 프로젝트 폴더의 `.env.example` 파일을 복사해 `.env` 로 이름을 바꾸고,
   메모장으로 열어 키를 입력 (키는 팀장에게 따로 받기):

   ```
   ANTHROPIC_API_KEY=sk-ant-...
   ```

> 터미널이 낯설면 **GitHub Desktop**([desktop.github.com](https://desktop.github.com))을 설치하세요.
> clone / pull / commit / push 가 전부 버튼 클릭입니다. (4번의 clone 도 GitHub Desktop 의
> "Clone a repository" 메뉴로 대체 가능)

---

## 2. 작업할 때마다 — 켜는 법

```bash
git pull        # ① 최신 상태 받기 (습관!)
npm run dev     # ② 서버 켜기
```

→ 브라우저에서 **http://localhost:3000/prompt-studio** 접속

끌 때는 터미널에서 `Ctrl+C`.

---

## 3. 화면 구성과 작업 흐름

**왼쪽 = 편집**

| 영역 | 내용 |
|---|---|
| 동료 페르소나 | NPC 5명의 이름·직업·성격. 탭으로 전환 |
| 연상 단어 프롬프트 | NPC가 접선 코드에서 단어를 떠올리는 규칙 ⚠ |
| 자유 대화 프롬프트 | NPC와 대화할 때의 말투·힌트 규칙 |

**오른쪽 = 미리보기** (저장 안 해도 지금 고친 초안 그대로 실행됨)

| 버튼 | 하는 일 | 소요 |
|---|---|---|
| 선택 NPC 1인 생성 | 코드 단어를 주고 이 NPC가 무슨 단어를 떠올리는지 | ~5초 |
| 5인 전원 생성 + 중복 판정 | 실제 게임과 동일하게 5명 생성 + 체포 판정까지 | ~20초 |
| 대화 미리보기 | 이 NPC에게 말 걸어보기 | ~3초 |

**추천 흐름**: 고치기 → 미리보기로 확인 → 마음에 들면 [저장] → 다음 판부터 게임에 적용.

---

## 4. ⚠ 규칙 3개 (꼭 읽기)

1. **자유롭게 만져도 되는 것** — 페르소나(성격·말투·배경), 자유 대화 프롬프트의 톤과
   힌트를 풀어주는 정도. 접선 코드(정답)는 이 프롬프트에 아예 주입되지 않는 구조라
   어떻게 고쳐도 정답이 유출될 수 없습니다.
2. **조심할 것 — 연상 단어 프롬프트.** "더 쉽게 / 더 직접적으로" 방향으로 고치면
   5명이 같은 단어로 수렴해 시작하자마자 다 잡혀가는 판이 됩니다 (실험으로 확인된 함정).
   고쳤으면 반드시 **[5인 전원 생성 + 중복 판정]을 코드 단어 2~3개로** 돌려보고,
   **체포 3명 이상이 반복되면 되돌리세요.** 평균 1~2명이 정상 범위입니다.
3. **금지** — NPC 의 `id` 변경 (게임 세션·스프라이트 연결이 깨집니다).

## 5. 작업 끝나면 — 올리기

터미널:

```bash
git add -A
git commit -m "tune: 에이다 말투를 더 무뚝뚝하게"   # 무엇을 했는지 한 줄
git push
```

GitHub Desktop: 왼쪽에 바뀐 파일 확인 → Summary 한 줄 쓰고 **Commit** → **Push origin**.

> 커밋 메시지에 남긴 튜닝 이력은 제출물(역할 기술서)에서 "누가 어떤 AI 파트를 만들었나"의
> 증거가 됩니다. 자잘하게 자주 커밋해도 좋아요.

---

## 6. 자주 묻는 것

- **미리보기 누르면 돈 나가?** — 실제 AI 호출이라 건당 몇 원 수준. 부담 없이 쓰되 무한 연타만 금지.
- **실수로 망쳤어** — 저장했어도 git 이 이력을 다 갖고 있어서 언제든 되돌릴 수 있습니다. 팀장에게 말하세요.
- **페이지가 403 이래** — `npm start` 로 켰기 때문. 스튜디오는 `npm run dev` 전용입니다.
- **로드 실패 / 접속 안 됨** — 터미널에 서버가 떠 있는지 확인 (`npm run dev` 다시 실행).
- **push 가 거부됨** — 팀장이 먼저 푸시한 게 있어서예요. `git pull` 한 번 하고 다시 `git push`.

## 7. 첫 튜닝 과제 (몸풀기 추천)

연상 단어 프롬프트의 규칙 1에 이런 문장이 있습니다:

> 너는 시계공이 아니라 {{role}}다.

그런데 에이다의 직업이 실제로 시계공이라, 에이다에게는 **"너는 시계공이 아니라 시계공다"** 로
렌더되는 어색한 문장이 됩니다. 이 문장을 직업과 무관하게 성립하는 표현으로 다듬고,
[5인 전원 생성]으로 결과가 여전히 좋은지 확인해 보세요. 도구 사용법 익히기에 딱 좋은 크기입니다.
