# 접선 코드 힌트 (글자 수·카테고리 사전 공개) — 설계

2026-07-21. "추리 범위가 너무 넓어 어렵다"는 난이도 문제에 대한 플레이어 쪽 처방.
스테이지 시작 시점부터 접선 코드의 **글자 수**와 **카테고리**(기계/장소/사물/자연)를 공개해
탐색 공간을 좁힌다. wordGen·중복 판정·경계 밸런스는 전혀 건드리지 않는 안전 레버다
(2026-07-18 `npm run exp:diff` 실측: wordGen 직접성 레버는 중복률 50%→75% 폭등 역효과).

## 이 세션에서 결정한 것

| 항목 | 결정 | 근거 |
|---|---|---|
| 노출 방식 | **시작 브리핑(접선 지령 쪽지) + 단서 수첩 상시 표시** | "범위가 너무 넓다"에 가장 직접적. 시작 시 1회 + 언제든 재확인 가능 |
| 튜토리얼 | **본편만 적용** | 튜토리얼은 고정 힌트로 이미 충분히 쉬움. 사서(t2)의 담당 축이 '분류'라 카테고리 힌트와 정면충돌 (사과 세트의 단서 「과일」이 허물이 됨) |
| 입력 검증 | **없음 — 정보로만** | 판정이 LLM 퍼지(애매하면 플레이어 유리)라 글자 수가 달라도 정답 처리될 수 있음 (예: 회중시계→시계). 차단·경고는 이와 모순 |
| 전달 경로 | **`toClientView`에 `hint` 필드** (A안) | 모든 상태 응답이 이 함수를 지나므로 클라이언트는 `state.hint` 하나만 보면 됨. 별도 필드/엔드포인트는 이득 없음 |
| 난이도 스위치 | 만들지 않음 — 항상 켬 | YAGNI. 스테이지별 온오프는 난이도 시스템이 생길 때의 향후 레버로만 기록 |

## 1. 서버 — `src/server/session.js`

`toClientView` 가 진행 중인 판에도 힌트를 포함한다:

```js
hint: {
  length: session.codeWord.length,   // 한글 음절은 BMP — .length 로 정확
  category: session.category,        // codewords.json 의 분류 키 그대로 (기계/장소/사물/자연)
},
```

**비유출 정책의 의도적 완화**다. 함수 위 주석을 갱신한다:
codeWord / reason 은 여전히 제외하되, 글자 수·카테고리는 힌트 기능으로 공개한다고 명시.
`createSession` 의 `category, // ← 서버 전용` 주석도 같이 갱신한다.

튜토리얼 세션(`tutorialSession.js`)은 손대지 않는다.

## 2. 클라이언트 — `src/client/scenes/StageScene.js`

**시작 브리핑** (접선 지령 쪽지, 현재 `StageScene.js:183` 부근):
"품 안에 조직이 남긴 쪽지" 프레이밍에 힌트를 서사로 싣는다. 쪽지 첫머리에 한 줄 추가:

> 코드는 3글자 — 기계 쪽 단어다.

**단서 수첩** (`#refreshClues`): 단서 유무와 무관하게 맨 위에 고정 줄을 표시한다:

> 접선 코드: ○○○ (기계)

○ 는 글자 수만큼 반복한다 (`'○'.repeat(hint.length)`).

빈 수첩(단서 0개)일 때도 힌트 줄은 보인다. `this.state.hint?.` 옵셔널 체이닝으로
필드 부재(구버전 응답 등)를 방어하고, 없으면 힌트 줄을 생략한다.

`TutorialScene` 은 수정하지 않는다.

## 3. 테스트 — `scripts/smoke-talk-sse.js`

start 응답 검증 추가:

- `state.hint.length` 가 1 이상의 정수 (풀에 1글자 코드 「재」가 있다)
- `state.hint.category` 가 `codewords.json` 의 4분류 중 하나
- 기존 `codeWord` 문자열 비유출 검사는 그대로 통과해야 함 (필드명 `hint` 라 충돌 없음)

## 4. 알려진 한계 (허용)

- NPC 연상 단어가 우연히 카테고리명과 같을 수 있다 (예: 코드 「톱니바퀴」 → 누군가 「기계」 제출).
  힌트가 그 단서를 겹치게 만들지만 체포 판정·밸런스에는 영향이 없다.
- 한국어 명사는 2글자가 압도적이라 글자 수 단독의 변별력은 약하다 — 카테고리와 묶여야 효과가 난다.
  (현재 풀은 1~4글자 혼재 — 1글자 「재」는 힌트만으로 후보가 급격히 좁아진다.)
- 힌트 셋(글자 수 + 카테고리 + 연상 단어)이 겹치면 쉬워질 수 있다. 지나치게 쉬워지면
  스테이지별 온오프 스위치가 다음 레버다 (이번 범위 아님).
